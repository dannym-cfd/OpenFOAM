Applied CFD, end to end

Three independent OpenFOAM projects — a GPU-accelerated race car external aero case, a supersonic missile aerodynamics case, and an automotive external aero case — each run from CAD geometry through meshing, solving, and force/moment extraction. These are working engineering case studies, not tutorials: real mesh-crash debugging, real solver pivots, and real numbers.

Race Car External Aero — GPU-Accelerated RANS/PIMPLE

Half-car symmetry-plane simulation at 50 m/s with vortex generators, underbody diffuser, engine intake flow, and a custom AmgX GPU pressure solve. Steady RANS baseline with a transient PIMPLE follow-up once the wake proved genuinely unsteady.

Cd 0.278 Cl -0.180 Solver simpleFoam / pimpleFoam + AmgX

Supersonic Missile Aerodynamics — Mach 3.8

A two-stage boosted missile at Mach 3.8115, 15.77 km altitude, sized from a custom flight-mechanics ascent trajectory. Two-mesh coarse-to-fine field-mapping workflow, 1.23M-cell final mesh, and an honest account of a GPU linear-solver stack that was built but not shipped.

Mach 3.81 Cd 0.031 Solver sonicFoam

Subaru Outback — Automotive External Aero

Custom-built vehicle geometry (Blender, then refined in Fusion 360) checked against a real trim-level build list, run through two design iterations. Drag coefficient improved substantially between geometry revisions.

Cd (rev1) 0.417 Cd (rev2) 0.352 Solver simpleFoam (inferred)
All three cases were built, meshed, and solved independently using OpenFOAM. Full technical detail — flow conditions, mesh strategy, debugging history, and results — is on each project's page.
